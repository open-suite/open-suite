# 3.1 — Registry images instead of node-local

- Phase: 3 (do first in phase; unlocks 3.2 and 3.6)
- Repos: open-suite/open-suite + open-suite/open-suite-portal (workflows)
- Depends on: 1.6 (org rename in `08`), 0.2/0.3/1.7a (auth-gate landed)
- Touches: `scripts/single-vps-deploy/08-open-suite-portal.sh`, `scripts/single-vps-deploy/12-auth-gate.sh`, portal repo `publish-images.yaml`
- Conflicts with: 3.2 (`08`), 4.5 (`08`), 0.3 (`12`)

## Change

Deploy pulls `ghcr.io/open-suite/*` images at pinned tags (CI already builds the frontend; add backend-overlay and auth-gate images to CI — auth-gate build context is `overlays/auth-gate/`). Remove on-box docker builds, `k3s ctr` imports, `:local` tags, `imagePullPolicy: Never`. Set the image via the env yaml/values so helmfile owns it, not `kubectl set image`.

## Verify

- Fresh deploy on a clean VM completes with no docker build step on the box.
- `kubectl get deploy -A -o yaml | grep -c 'imagePullPolicy: Never'` is 0.
- Pods survive `helmfile apply` with the right image.
