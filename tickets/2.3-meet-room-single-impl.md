## Status 2026-07-02 — blocked, premise false

Attempted and reverted. `POST /apps/meetcal/room` authenticates fine with
johndoe's app password (verified in-cluster), but returns 401 `no_token`:
user_oidc keeps the OIDC login token in the interactive PHP session only
(`TokenService::SESSION_TOKEN_KEY`), so a non-session request can never mint
the exchange token. The seed would need a real browser-style OIDC login flow
(gate + Keycloak form) to get a session cookie. Options if this is revisited:
(a) script the full OIDC login in the seed, (b) have meetcal fall back to a
client-credentials/service-account exchange for non-session callers, or
(c) accept the duplicate bash implementation. The public route is also behind
the auth gate — in-cluster calls bypass it.
