## Change

Express the nginx sidecar + targetPort change as a patch to the vendored charts (or a values-level extraContainers hook where charts expose one), keeping only the ConfigMap upload in the script. Remove the namespace-wide blind `rollout restart ... || true` (line 41) in favour of per-deployment `rollout status` waits; replace `/spec/ports/0/targetPort` index addressing with name-based patching.

## Status 2026-07-02 — partially done

Landed the script-hygiene half: `patch_static` restarts exactly the
deployments that mount each configmap (verified names on the box:
meet-frontend, meet-static-nginx, element-web) with per-deployment
`rollout status` waits, and the Service reroute targets the `http` port
by name instead of `/spec/ports/0`.

The declarative half remains. Findings: all four charts (nextcloud,
grist, docs, bureaublad) support a `sidecars` values hook, but every
Service template hardcodes `targetPort: {{ .Values.*.containerPorts.http }}`
— so the sidecar reroute needs a chart patch introducing a targetPort
override, plus env-values plumbing for the per-app nginx.conf, and its
acceptance test is a live `helmfile apply` on the demo box (which today
breaks the header — the regression this ticket exists to fix). Do that
as its own change with a maintenance window on the demo.

## Status 2026-07-03 — done

Declarative half landed: `patches/local/opensuite-header-sidecar.patch` adds
to the four vendored charts/values — the nginx sidecar (`sidecars` hook),
`extraContainerPorts` 8091 (which also opens the chart NetworkPolicies; the
bureaublad frontend netpol needed a hunk to honour it), a
`service.targetPortOverride` introduced in each service template, the
`opensuite-header-conf` ConfigMap via `extraDeploy` (docs and bureaublad
charts needed an `extra-list.yaml` added), and an optional dir-mounted
`opensuite-header-js` configmap. 09 now only uploads that JS configmap
(kubelet syncs the dir mount, no restart) and patches the static SPAs.
Cache-busting moved from ?v=<hash>+immutable to no-cache revalidation, so
the conf is fully static. Verified live 2026-07-03: helmfile sync produces the sidecar, reroute and netpol on all four apps; the authenticated smoke shows the header on nextcloud/grist/docs after repeated syncs. Follow-ups landed the same day: #67 duplicate port, #69/#70 extraIngress 8091 for the from-restricted netpols (grist/nextcloud 502 otherwise), #71 image/env pins into values (a helm sync had reverted the kubectl-patched portal images — same failure class this ticket exists to end), #72 meet registry decouple.
